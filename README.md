# AI Resume Analyzer

Upload a resume, paste a job description, and get an ATS score with AI-powered breakdowns.

**Stack:** Node.js, Express, OpenAI (GPT-4o-mini), Redis, Docker

## Features

- **ATS Scoring** — overall score (0–100) with sub-scores for keywords, skills, experience, education, format
- **Keyword Analysis** — matched vs missing keywords against the job description
- **AI Suggestions** — actionable improvements generated by GPT-4o-mini
- **Resume Parsing** — auto-extracts contact info, skills, experience, education from PDF/DOCX
- **Redis Caching** — AI results cached 24h to save API costs and speed up repeat analyses
- **Rate Limiting** — Redis-backed: 100 req/15min global, 10 analyses/hour per IP
- **Security** — Helmet, CORS, input validation, non-root Docker user
- **Graceful Degradation** — runs without Redis (falls back to in-memory rate limiting, no caching)

## Quick Start (Local)

```bash
# 1. Clone and install
npm install

# 2. Set your OpenAI key
cp .env.example .env
# Edit .env → set OPENAI_API_KEY

# 3. Start Redis (optional — app works without it)
redis-server

# 4. Run
npm start
# → http://localhost:3000
```

## Docker (Production)

```bash
# Set your key as an env variable or in .env
export OPENAI_API_KEY=sk-...

# Build & start (app + redis)
docker compose up --build -d

# Stop
docker compose down
```

## Environment Variables

| Variable | Required | Default | Description |
|---|---|---|---|
| `OPENAI_API_KEY` | **Yes** | — | OpenAI API key ([get one here](https://platform.openai.com/api-keys)) |
| `PORT` | No | 3000 | Server port |
| `REDIS_URL` | No | `redis://localhost:6379` | Omit or leave empty to disable Redis |
| `CORS_ORIGIN` | No | `*` | Lock to your domain in production (e.g. `https://example.com`) |
| `LOG_LEVEL` | No | `info` | `debug` for verbose logging |
| `NODE_ENV` | No | `development` | Set `production` to enable JSON logs, file logging, and static caching |

## API

### `POST /api/analyze`
Analyze a resume against a job description.

**Request:** multipart/form-data
- `resume` — PDF or DOCX file (max 5MB)
- `jobDescription` — string

**Response:**
```json
{
  "atsScore": 78,
  "keywordMatch": { "score": 72, "matched": ["React", "Node.js"], "missing": ["Kubernetes"] },
  "formatScore": 90,
  "experienceScore": 75,
  "educationScore": 80,
  "skillsScore": 70,
  "strengths": ["Strong backend experience"],
  "improvements": ["Add more quantifiable achievements"],
  "suggestions": ["Include a professional summary section"],
  "summary": "The resume is well-structured but lacks several key skills.",
  "sections": { "contact": true, "summary": false, "experience": true, "education": true, "skills": true, "certifications": false }
}
```

### `GET /api/health`
```json
{ "status": "ok", "uptime": 123.45, "redis": true }
```

## Deployment

### Single VPS
```bash
git clone <repo> /opt/resume-analyzer
cd /opt/resume-analyzer
docker compose up --build -d
```
Put behind nginx/caddy with SSL. Set `CORS_ORIGIN` to your domain.

### Railway / Render / Fly.io
- These platforms auto-detect the `Dockerfile`
- Add a managed Redis addon and set `REDIS_URL`
- Set `OPENAI_API_KEY` and `NODE_ENV=production` in the dashboard

### No Redis? No problem.
The app runs fully without Redis — rate limiting uses in-memory storage and caching is skipped. Just set `REDIS_URL` to an empty value or omit it.
